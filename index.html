Technologies

Extending past the standard Android application, In Rotation was built on many current and powerful technologies. Some of them include:

  • Spotify SDK and Web API
  • Firebase Real-Time Database for quick updates to active playlists and users in the system
  • Volley HTTP Request Library

Design Patterns Used

To achieve high performance and a foundation of understandable code, popular design patterns were used in different components of the system. They are as follows (in the current build):

  • Singleton Pattern for the Spotify Profile Access and Playback functionality

    Benefit: Access to interact with music player and current playlist

  • Abstract Factory Pattern for creating collaborative playlists

    Benefit: Playlists can be made at runtime with filling up the database

  • Observer Pattern for notifying the playlist view when the database updates

    Benefit: The user does not have to refresh the playlist to get new songs

The One Thing I like best

The thing I like best about my design is that I kept the playlist implementation separate from Spotify's implementation. This allows me to have more control and customization of the playlist structure and offer the possibility of authenticating with different services (i.e. SoundCloud, Apple Music, Google Play Music).

Key Learning Experience

The key thing I learned throughout this project was that trying to develop understandable code is very difficult at first, but helps immensely when trying to implement more complex functionality and a more robust system.

Another key learning experience was learn the fact that when you are hours into an implementation of some functionality, it is almost always aguaranteed that you will think of a better/more efficient way to do it.
